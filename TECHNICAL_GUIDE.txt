1. アーキテクチャ概要
================================================================

データの流れ：

  iRacing SDK（メモリマップドファイル）
      ↓
  TelemetryService / IbtService
      SDKから生データを読み取り TelemetrySnapshot に格納
      ↓
  SlipAngleCalculator / FfbAnalyzer
      スナップショットを受け取り物理計算を実施
      SlipAngleResult（FFBフィールド含む）を生成
      ↓
  MainViewModel → OverlayViewModel / FfbOverlayViewModel
      計算結果をUI用の文字列・グラフデータに変換
      ↓
  OverlayWindow / FfbOverlayWindow（WPF）
      透過オーバーレイとしてゲーム画面に表示

・ライブ接続：iRacing SDK のメモリマップドファイルを 60Hz でポーリング
・IBT再生  ：.ibt バイナリファイルをフレーム単位で読み取り同じパイプラインで処理

  FFB（フォースフィードバック）    360 Hz

4. スリップアングル計算の詳細
================================================================

■ 4.1 基本スリップ角

  iRacing SDKの速度ベクトルはSDKアンテナ基準点（車体中央付近、
  実際のCoGとは異なる位置）で測定されています。

  基本スリップ角 = atan2(VelocityY, VelocityX)

■ 4.2 剛体速度変換（SLIP VECTOR TRANSLATION）

  車体は剛体なので、任意の点での速度は以下の式で求められます。

    Vx_new = VelocityX − YawRate × b
    Vy_new = VelocityY + YawRate × a

    a = 横方向オフセット（右が正）
    b = 前後方向オフセット（前が正）

    スリップ角 = atan2(Vy_new, Vx_new) × (180 ÷ π)  [度]

  この計算を CoG・FL・FR・RL・RR の各点に適用します。

■ 4.3 タイヤスリップ角の計算

  タイヤスリップ角 = 操舵角 − ホイール位置のスリップ角

  フロントホイールにはアッカーマンジオメトリを適用します
  （コーナリング時、内輪と外輪で操舵角が異なるため）。

  リアホイールの操舵角は 0°（標準的な後輪非操舵車を前提）。

5. 左右・前後スリップアングルに差が出る原因
================================================================

左右または前後のスリップ角に差が生じる場合、以下の原因が考えられます。

■ 原因1：CoGオフセット（横方向）

  CoGOffsetFromRef_A（横方向オフセット）がゼロでない場合、
  右ホイールと左ホイールへの速度変換式が非対称になります。

    右後輪への Vy_new = VelocityY + YawRate × (CoGOffset_A + TrackWidth/2)
    左後輪への Vy_new = VelocityY + YawRate × (CoGOffset_A − TrackWidth/2)

  CoGOffset_A がゼロでない場合、左右のスリップ角が系統的にずれます。

  現在のデフォルト値は 0.0 に設定されており、この原因は通常発生しません。
  左右差が大きい場合は Vehicle Parameters で CoGOffsetFromRef_A を
  確認してください。

■ 原因2：CgFrontRatio の精度（前後方向）

  CgFrontRatio は前後軸のCoGからの距離を決定します。
  carspec.json に登録されていない車両ではデフォルト値（0.45）が使われ、
  実際の重心位置と異なる場合に前後スリップ角に誤差が生じます。

  例：リアエンジン車（ポルシェ等）で CgFrontRatio = 0.45 のまま使用すると
  フロントのCoG距離が実際より長く、リアが短く計算される。

  対処法：Vehicle Parameters の Car DB 欄を確認し、車両が正しく
  認識されているかを確認してください。

■ 原因3：低速時の atan2 感度増大

  低速では VelocityX が非常に小さくなるため、わずかな VelocityY の
  ノイズが大きなスリップ角として現れます。

    速度 100 km/h：VelocityX ≈ 27.8 m/s  → ノイズ感度：低
    速度   5 km/h：VelocityX ≈  1.4 m/s  → ノイズ感度：高（約20倍）

  速度しきい値フィルターを適切に設定することで低速ノイズを抑制できます。

■ 原因4：アッカーマンジオメトリ（フロントのみ）

  低速旋回では内輪と外輪で操舵角が大きく異なります。
  コーナーの向きによって右前・左前のどちらかが大きなスリップを示します。
  これは計算誤差ではなく実際の幾何学的特性です。

■ 原因5：路面カント・デフ動作

  ・iRacing の路面は排水のため横方向に傾斜しており荷重が偏ることがある
  ・LSD（リミテッドスリップデフ）のトルクバイアスにより
    左右後輪に差が出る車種が存在する

■ 原因6：サーキットのコーナー方向偏り

  走行サーキットが特定の方向のコーナーを多く含む場合、
  外輪となる側のタイヤは荷重・スリップともに大きくなります。
  これは計算誤差ではなく実際の物理現象です。

6. フィルター機能の解説
================================================================

スリップ角計算には低速時のノイズが乗りやすいため、
2種類のフィルターを実装しています。
コントロールパネルの「Filter Settings」タブから調整できます。

■ 6.1 速度しきい値フィルター

  役割：設定速度以下では全スリップ角の計算を無効化（ゼロ表示）します。

    Speed < しきい値 → スリップ角 = 0

  設定値の目安：
    低い値（例：5 km/h）   ほぼすべての走行データを表示
    高い値（例：30 km/h）  低速ノイズを完全にカット

  理由：速度が低いと VelocityX が非常に小さくなり、
  atan2 の計算結果が不安定になります。

■ 6.2 指数移動平均（EMA）フィルター

  役割：フレームごとのスリップ角の急激な変動を滑らかにします。

    フィルター後 = α × 今回の値 + (1 − α) × 前回のフィルター後の値

  α（アルファ）の値による違い：
    α = 1.0          フィルターなし（生データそのまま）
    α = 0.5          中程度の平滑化、応答遅延は小さい
    α = 0.3（推奨）  バランス良好
    α = 0.1          強い平滑化、ノイズは消えるが遅延が増える

  コントロールパネルのEMAスライダー（0〜1）でリアルタイムに調整可能。

■ 6.3 G-force 専用フィルター（参考）

  G-forceは速度の時間微分で求めるため、6サンプル分の単純移動平均を
  別途適用しています。コントロールパネルからは変更できません。

7. パネルの説明
================================================================

■ SLIP VECTOR TRANSLATION
  各ホイール位置のスリップ角を数値で表示します。
  値が大きいほど、そのタイヤが横方向に滑っている（または向きがずれている）
  ことを示します。コーナリング中は外輪側が大きくなるのが正常です。

  【荷重移動バー】
  各タイヤ位置の外側に縦バーで荷重変化をリアルタイム表示します。
  データソース：ShockDefl（サスペンション変位量）−ベースライン値

  フロント（左右）：
    上に伸びる（オレンジ）  荷重増（サスペンション圧縮）
    下に伸びる（ブルー）    荷重減（サスペンション伸長）

  リア（左右）：
    下に伸びる（オレンジ）  荷重増（サスペンション圧縮）
    上に伸びる（ブルー）    荷重減（サスペンション伸長）

  ※ フロントとリアでバーの伸びる方向が逆になっています。
    ブレーキング時に「前バー上オレンジ・後ろバー上ブルー」、
    加速時に「前バー下ブルー・後ろバー下オレンジ」と表示され、
    荷重の移動方向を直感的に把握できます。

  活用例：
    ブレーキング  → 前輪: 上オレンジ（荷重増）、後輪: 上ブルー（荷重減）
    加速          → 前輪: 下ブルー（荷重減）、後輪: 下オレンジ（荷重増）
    左コーナー    → 右側2輪: オレンジ（外輪荷重増）、左側2輪: ブルー（内輪荷重減）

  荷重ベースラインは走行開始時の低速（< 5 km/h）フレームで自動設定。

  表示対応モード：
    ライブ走行      60Hz リアルタイム
    ANALYSISモード  ガレージリプレイ中に走行ログから再現（ライブ走行と同精度）
    IBT再生         IBTファイルの ShockDefl から表示
    保存リプレイ    iRacing仕様上 ShockDefl が記録されないため非表示

■ REALTIME TELEMETRY グラフ
  フロント平均・リア平均・Balance（F−R）スリップ角の時系列グラフです。
  横軸が時間（設定したウィンドウ幅）、縦軸がスリップ角（°）です。

■ PEDAL INPUT グラフ
  アクセル開度（緑）・ブレーキ開度（赤）・クラッチ開度（オレンジ）の
  時系列グラフです。各ラインはボタンで表示オン/オフ切替可能です。

■ SDK RAW DATA
  iRacing SDKから直接取得した生テレメトリー値を表示します。
  計算処理を一切行わない生データです。

■ G-FORCE（トラクションサークル）
  SDK の LatAccel / LongAccel から算出した前後左右のG値を
  円グラフ上の点で表示します。白い軌跡で直近10秒の動きを確認できます。
  ABS 作動時はオレンジに点滅します。

■ FFB GRIP ANALYSIS
  iRacing SDK の SteeringWheelTorque（ステアリングトルク = FFB 信号）を
  元にタイヤのグリップ状態をリアルタイム解析します。

  表示項目：
    FFB Raw       現在の生FFB値（N·m）
    FFB Filtered  EMA平滑化後のFFB値
    FFB Peak      動的ピーク学習値（走行中の最大FFBを追跡・decay=0.9999/frame）
    GripUtil      グリップ使用率 = FFB_filtered / FfbPeak（0〜100%）
    SlipEff       スリップ効率（グリップ利用度の効率指標）

  状態バッジ：
    NORMAL      グリップに余裕あり
    NEAR LIMIT  グリップ限界付近
    OVER LIMIT  限界超え / スナップオーバーステア検知

  スナップオーバーステア検知：
    ヒステリシス付きの状態マシンによりスナップ発生を検知します。
    検知時は OVER LIMIT バッジが3サイクル点滅して警告します。

  GripUtilization 時系列グラフ（Y軸 0〜100%）付き。

  注意：セッション開始直後はFfbPeakが未学習のため GripUtil が高く
  表示されます。数十秒の走行でピーク値が安定します（ウォームアップ期間）。

  表示対応モード：
    ライブ走行      60Hz リアルタイム
    ANALYSISモード  ガレージリプレイ中に走行ログから再現
    保存リプレイ    FFB値が記録されないため非表示

8. Vehicle Parameters（車両パラメーター）
================================================================

スリップ角計算の精度に直接影響するパラメーターです。
コントロールパネルの「Vehicle Parameters」タブで現在の設定値を
確認できます。

■ 8.1 carspec.json（外部車両DB）から自動取得するパラメーター

  アプリ起動時に GitHub 上の carspec.json を取得し、
  iRacing 接続時の CarPath と照合して車両データを自動設定します。
  車両を変更するたびに自動更新されます。

  取得元の優先順位：
    1. GitHub（https://raw.githubusercontent.com/.../carspec.json）
    2. インストールフォルダ内の carspec.json（オフライン時）
    3. アプリ内蔵デフォルト値

  carspec.json から取得するパラメーター：

    Wheelbase          ホイールベース（前後軸間距離）     単位: m
    Track Width Front  フロントのトレッド幅（左右接地面間） 単位: m
    Track Width Rear   リアのトレッド幅                   単位: m
    CgFrontRatio       重心前後位置比率（後述 8.2 参照）   単位: 無次元
    SteeringRatio      ステアリングギア比（ハンドル角 ÷ タイヤ角）

  コントロールパネルの「Car DB」欄に一致した車両名とデータ取得元が
  表示されます。一致しない場合は「デフォルト値 (CarPath: xxx)」と
  表示されます。その場合は CarPath を開発者に報告してください。

■ 8.2 CgFrontRatio（重心前後位置比率）

  「フロント車軸から重心（CoG）までの距離」÷「ホイールベース」

    前軸 ←── CgFrontRatio × WB ──→ CoG ←── (1-CgFrontRatio) × WB ──→ 後軸

  値の解釈：
    0.50  → CoGが前後軸の中央（MX-5 Cup など）
    0.45  → CoGは前軸から45%、後軸から55%の位置（標準的なFRスポーツ）
    0.435 → CoGが後寄り（ポルシェ 911 GT3 R 992 など RR/MR 系）

  実例：ポルシェ 911 GT3 R (992)（CgFrontRatio = 0.435、WB = 2.507m）
    前軸から CoG まで = 2.507 × 0.435 = 1.091 m
    CoG から後軸まで = 2.507 × 0.565 = 1.416 m
    → リアエンジンのため重心は後方寄りになる

  実例：MX-5 Cup 2016（CgFrontRatio = 0.50、WB = 2.311m）
    前軸から CoG まで = 2.311 × 0.50 = 1.156 m
    CoG から後軸まで = 2.311 × 0.50 = 1.156 m
    → 前後ほぼ均等な重量配分

  スリップ角計算への影響：
    各タイヤのCoGからの前後距離を決定し、ヨーレート補正の精度に
    直結します。値が不正確だと前後タイヤのスリップ角に誤差が生じます。
    特にリア重心の車両（ポルシェ等）では、0.45（汎用デフォルト）との
    差が大きいため、正確な値の設定が重要です。

    フロント左タイヤの補正速度計算例（CgFrontRatio = 0.45, WB = 2.65m）：
      前軸～CoG距離 = 2.65 × 0.45 = 1.193 m
      Vx_FL = VelocityX - YawRate × 1.193
      Vy_FL = VelocityY + YawRate × (CoGOffsetA - TrackWidthFront/2)

■ 8.3 CoGHeight（重心高さ）の自動設定

  CoGHeight は以下の優先順位で設定されます。

  【優先順位 1】iRacing SDK 実測値（IRSDKSharper v1.1.6以降）
    SDK が重心高さを公開している場合は実測値を自動取得します。
    コントロールパネルの「車両カテゴリー（自動判定）」欄に
    「（SDK実測値）」として反映されます。

  【優先順位 2】カテゴリー別デフォルト値
    SDK から取得できない場合、carspec.json のカテゴリーに基づいて
    以下のデフォルト値を使用します。

    カテゴリー                       CoGHeight（重心高）     根拠
    ──────────────────────────────────────────────────────────
    GT3 / GT4 / GTP                   0.42 m               標準的なGTカー
    LMP2 / LMP3                       0.32 m               低重心プロトタイプ
    Formula / IndyCar                  0.28 m               最低地上高のフォーミュラ
    TCR / Touring / Cup / Sports       0.45 m               量産ベースの高め重心
    汎用（判定不能）                   0.42 m               GTカー相当

  ※ 荷重移動バーは ShockDefl（サスペンション変位量）を直接使用しており、
    CoGHeight とは独立しています。

■ 8.4 SDK非公開のため固定値を使用するパラメーター

  iRacing SDK は速度計測基準点の正確な位置を公開していないため、
  以下の値は経験的な固定値を使用しています。

  CoGOffsetFromRef_A（横方向オフセット） = 0.0 m
  CoGOffsetFromRef_B（前後方向オフセット） = -1.0 m

  CoGOffsetFromRef_B = -1.0 m の意味：
    iRacing SDK の VelocityX/Y が計測される基準点が、
    CoG より約 1m 前方にあると推定した値です。
    iRacing が基準点の位置を非公開のため、実測値ではなく
    動作比較による経験値です。
    ※ CgFrontRatio（CoG と各軸の距離）とは独立した別の概念です。

  CoGOffsetFromRef_A = 0.0 m の意味：
    横方向のオフセットはゼロと仮定しています。
    非ゼロにすると左右のスリップ角が系統的に非対称になります。

■ 8.5 各パラメーターのスリップ角計算への影響まとめ

  パラメーター            影響する計算
  ──────────────────────────────────────────────────────────────
  Wheelbase             前後ホイールのCoGからの距離（前後補正）
  Track Width F/R       左右ホイールのCoGからの距離（左右補正）
  CgFrontRatio          前後補正量の比率（重心が前寄り/後ろ寄り）
  SteeringRatio         ロードホイール角 = ステアリング舵角 ÷ SR
  CoGOffsetFromRef_A/B  全ホイールに加算されるSDK基準点補正
  CoGHeight             参照表示のみ（荷重移動バーは ShockDefl を使用）

  速度変換式：
    Vx_wheel = VelocityX - YawRate × (CoGOffsetFromRef_B + 前後オフセット)
    Vy_wheel = VelocityY + YawRate × (CoGOffsetFromRef_A + 横オフセット)
    スリップ角 = atan2(Vy_wheel, Vx_wheel) [度換算]


================================================================
9. スリップアングルの適正値
================================================================

車種・タイヤ・走行条件によって異なりますが、一般的な目安は以下の通りです。

■ フロントタイヤ（コーナリング中）

  グリップ限界以下（安定走行）        3° 〜  8°
  ピークグリップ（最適）              6° 〜 12°
  オーバースリップ（アンダーステア）  12° 以上

■ リアタイヤ（コーナリング中）

  グリップ限界以下（安定走行）        2° 〜  6°
  ピークグリップ（最適）              4° 〜 10°
  オーバースリップ（オーバーステア）  10° 以上

■ 車種別の傾向

  フォーミュラカー            フロント  3°〜 6°  リア  2°〜 5°
  GT / スポーツカー           フロント  6°〜10°  リア  4°〜 8°
  スポーツカー（ソフトタイヤ） フロント  8°〜12°  リア  6°〜10°

■ ドライビング判断の目安

  フロント > リア          → アンダーステア傾向
  リア > フロント          → オーバーステア傾向
  前後バランスが取れている  → ニュートラルステア（理想）

■ 重要な注意点

  このアプリが表示しているのは車体スリップ角（Body Slip）を元に
  計算した値であり、タイヤ接地面での純粋なスリップ角とは厳密には
  異なります。上記の数値はあくまで参考値として使用し、実際の挙動
  （アンダー・オーバーの感覚）と照らし合わせて判断することを推奨します。

10. ANALYSISモード・保存リプレイについて
================================================================

■ 10.1 ANALYSISモード（ガレージリプレイ）

  iRacing でコースを走り終えてガレージに戻ると、iRacing が走行を自動的に
  リプレイします（EngineWarnings の ReplayProcess ビットがON）。
  このとき iSAV は ANALYSISモード に切り替わります。

  データソース：走行中にリアルタイム記録した内部ライブログ（LiveLog）
  記録内容   ：SessionTime / スリップ角 / ペダル入力 / ShockDefl 4輪 / FFB / etc.
  同期方法   ：SDK の ReplaySessionTime でログを索引して各フレームを参照

  表示できるもの：
    ・スリップ角グラフ（走行の完全再現）
    ・ペダル入力グラフ（同上）
    ・荷重移動バー（ShockDeflをログから再現。ライブ走行と同精度）
    ・FFB Grip Analysis（FFBログから再現）

  注意：ライブ走行データのないセッション開始直後はログが空のため
  表示されません。

■ 10.2 保存リプレイ（Saved Replay / .rpy）

  iRacing の「Saved Replays」メニューから .rpy ファイルを再生する場合、
  iRacing の仕様により ShockDefl が全輪ほぼ同一値で記録されているため
  荷重移動バーは非表示になります。

  この制約を回避するため、ROLLANGLE（ロール角）に以下の高精度計算を適用します。

  【適応バンク補正（Adaptive Bank Correction）】
    直線区間（|横G| < 0.5 m/s²）を自動検出し、路面バンク角をEMAで推定します。
    α = 0.003 の EMA により緩やかにバンク角に追従します。
    これにより「路面バンクによる常時ロール」を差し引いた純粋な
    サスペンションロールを算出します。

  【自動K係数キャリブレーション（忘却型最小二乗法）】
    コーナリング中（|横G| ≥ 2.0 m/s²）のデータを蓄積し、
    横G（LatAccel）とロール角（RollRad − バンク推定値）の線形関係から
    係数 K を自動計算します（忘却係数 0.99 の忘却型最小二乗法）。

    キャリブレーション完了後：
      ROLLANGLE ≈ LatAccel × K（バンク影響を除いた純粋なサスペンションロール）

    未完了時：
      ROLLANGLE = RollRad − 適応バンク推定値

  キャリブレーション状態は DIAGNOSTICS パネルで確認できます。
    cal:True  → キャリブレーション完了（LatAccel × K を使用）
    cal:False → 未完了（適応バンク補正のみ）

  通常、数十秒のコーナリングデータで cal:True に切り替わります。
